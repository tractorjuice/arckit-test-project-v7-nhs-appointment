Features

Systematic

Follow GDS Agile Delivery phases (Discovery → Alpha → Beta → Live) with structured command workflow ensuring complete, auditable project artifacts.

Compliant

Built-in compliance with Technology Code of Practice (TCoP), Secure by Design principles, GDS Service Standard, and UK Government procurement frameworks.

Auditable

Every artifact includes version control, traceability matrices, design review checklists, and comprehensive gap analysis for audit readiness.

AI-Assisted

Works with Claude Code, OpenAI Codex CLI, and Gemini CLI. AI generates first drafts following templates; architects review and refine.

25 ArcKit Commands

Organized by GDS Agile Delivery phase for systematic project execution:

Phase 0: Planning

  • /arckit.plan - Create comprehensive project plan with GDS phases, Mermaid Gantt charts, and workflow diagrams

Discovery Phase

  • /arckit.stakeholders - Analyze stakeholder drivers and goals (RACI matrix, power/interest grid)
  • /arckit.risk - Create comprehensive risk register (technical, commercial, operational risks)
  • /arckit.sobc - Generate Strategic Outline Business Case (5-case model: Strategic, Economic, Commercial, Financial, Management)

Alpha Phase

  • /arckit.requirements - Define comprehensive requirements (Business, Functional, Non-Functional, Data, Integration)
  • /arckit.arch-principles - Establish architecture principles (Security, Performance, Scalability, Interoperability)
  • /arckit.data-model - Design entity-relationship data model with GDPR compliance
  • /arckit.wardley - Create Wardley Map for strategic planning and build vs buy decisions
  • /arckit.sow - Generate Statement of Work for vendor procurement
  • /arckit.evaluation-criteria - Define vendor evaluation scoring criteria
  • /arckit.compliance - Map compliance requirements (GDPR, Accessibility, Security standards)

Beta Phase

  • /arckit.hld - Generate High-Level Design (system architecture, technology stack, integration patterns)
  • /arckit.lld - Create Low-Level Design (detailed component design, APIs, data flows)
  • /arckit.diagram - Generate architecture diagrams (C4 Model: Context, Container, Component, Code)
  • /arckit.test-strategy - Define testing approach (unit, integration, E2E, performance, security)
  • /arckit.traceability - Create traceability matrix linking requirements → design → implementation → tests
  • /arckit.runbook - Generate operational runbook (deployment, monitoring, incident response)
  • /arckit.analyze - Comprehensive gap analysis across all project artifacts

UK Government Procurement

  • /arckit.dos - Digital Outcomes and Specialists framework guidance
  • /arckit.gcloud - G-Cloud framework guidance and requirements

Design Reviews

  • /arckit.design-review - Comprehensive design review checklist (10-section assessment)
  • /arckit.hld-review - High-Level Design review with scoring criteria
  • /arckit.lld-review - Low-Level Design review with technical assessment

Utilities

  • /arckit.research - Internet research with cited sources for technical decisions
  • /arckit.init - Initialize ArcKit in a new repository
  • /arckit.help - Display all available commands with descriptions

Multi-AI System Support

ArcKit works with three major AI development environments:

Claude Code

Primary

Official CLI from Anthropic. Commands in .claude/prompts/ directory.

Best for: Complex reasoning, architecture design, comprehensive documentation generation.

Installation: brew install claude

OpenAI Codex CLI

Supported

Codex CLI for GPT-4. Commands in .codex/prompts/ directory.

Best for: Code generation, technical implementation, API integration.

Installation: npm install -g @openai/codex-cli

Gemini CLI

Supported

Google Gemini CLI. Commands in .gemini/prompts/ directory (TOML format).

Best for: Multi-modal analysis, data processing, rapid prototyping.

Installation: pip install gemini-cli

All commands work identically across AI systems. Choose based on your organization's AI platform preference.

Getting Started (5 Minutes)

1. Install Claude Code

brew install claude

2. Clone ArcKit

git clone https://github.com/tractorjuice/arc-kit.git cd arc-kit

3. Initialize in Your Project

cd /path/to/your/project claude /arckit.init

4. Run Your First Command

/arckit.plan Create project plan for NHS appointment booking system

5. Follow GDS Phases

# Discovery /arckit.stakeholders Analyze stakeholders for appointment system /arckit.risk Create risk register /arckit.sobc Generate Strategic Outline Business Case # Alpha /arckit.requirements Define requirements /arckit.arch-principles Establish architecture principles /arckit.wardley Create Wardley Map for build vs buy decisions # Beta /arckit.hld Generate High-Level Design /arckit.diagram Create C4 architecture diagrams /arckit.analyze Comprehensive gap analysis
! Warning AI generates first drafts following templates. Always review, validate, and refine AI output with human expertise.
